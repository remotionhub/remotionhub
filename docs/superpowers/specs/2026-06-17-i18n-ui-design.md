# RemotionHub UI i18n Design

## Goal

Add UI internationalization for RemotionHub with Chinese and English support.

Default language is Chinese. Users can switch between Chinese and English from the header, and the preference is persisted in localStorage.

## Scope

This design covers platform UI text only.

Catalog data is not translated by the platform. Component titles, summaries, changelogs, usage markdown, agent prompts, and source metadata are displayed exactly as uploaded/imported from catalog data.

No Convex catalog schema changes are required.

## Non-Goals

- No URL locale prefix such as `/zh/remotion` or `/en/remotion`.
- No `?lang=` query parameter.
- No server-side language negotiation.
- No cookie-based locale persistence.
- No translated catalog fields in `Component`, `ComponentVersion`, or `Artifact`.
- No translation workflow for user-uploaded catalog content.

## Locale Model

Supported locales:

- `zh`
- `en`

Default locale:

- `zh`

Persistence key:

```text
remotionhub.locale
```

Invalid or missing persisted values fall back to `zh`.

## Architecture

Add a thin frontend-only i18n layer:

- `src/lib/i18n.ts`
  - Defines `Locale`.
  - Defines supported locales.
  - Defines translation dictionaries.
  - Provides a typed `TranslationKey`.
  - Provides `resolveLocale`.
  - Provides `translate`.

- `src/components/I18nProvider.tsx`
  - Initializes locale to `zh`.
  - On client mount, reads `localStorage['remotionhub.locale']`.
  - Exposes `locale`, `setLocale`, and `t`.
  - Writes locale changes back to localStorage.
  - Handles localStorage failures defensively.

- `src/components/LanguageToggle.tsx`
  - Header control with two choices: `中文` and `EN`.
  - Calls `setLocale('zh')` or `setLocale('en')`.
  - Shows current active locale.

- `src/components/AppProviders.tsx`
  - Wraps app content in `I18nProvider` inside the existing Convex provider.

Consumers use a hook:

```ts
const { t, locale, setLocale } = useI18n()
```

## Metadata Behavior

Route metadata remains Chinese by default in the MVP.

Because this design does not use locale in URL, cookies, or server negotiation, SSR metadata cannot reliably know the user's localStorage preference. Client-visible page copy can switch languages after hydration, but metadata generated during SSR should use Chinese defaults.

Detail route metadata still uses catalog data as-is:

- Component display name.
- Component summary.

Catalog metadata is not translated.

## Data Flow

1. `I18nProvider` initializes locale state to `zh`.
2. In the browser, `I18nProvider` reads localStorage.
3. If stored locale is `zh` or `en`, provider updates state.
4. User clicks `中文` or `EN` in `LanguageToggle`.
5. `setLocale` updates React state and writes localStorage.
6. Components using `useI18n().t(key)` re-render.
7. Catalog data fields continue to render directly from Convex query results.

## Error Handling

- If localStorage read throws, keep `zh`.
- If localStorage write throws, update in-memory locale but do not crash.
- If persisted locale is invalid, use `zh`.
- If a translation key is missing:
  - In development, throw an error or surface the missing key clearly.
  - In production, return the key string to avoid crashing the app.

## Testing

Unit tests:

- Default locale is `zh`.
- Invalid locale resolves to `zh`.
- English translation returns expected text.
- Chinese translation returns expected text.
- Dictionaries have identical key sets.

Component tests:

- Header renders Chinese by default.
- Clicking `EN` changes visible navigation text to English.
- Clicking `中文` changes visible navigation text back to Chinese.
- Language preference is written to localStorage.

Playwright smoke tests:

- Default browse page renders Chinese UI text.
- Runtime pages render catalog data unchanged.
- Detail page renders catalog data unchanged and UI labels in Chinese by default.
- Switching to English changes UI labels without changing catalog titles or summaries.

Existing Convex contract tests should not change because the backend data model is unchanged.

## Acceptance Criteria

- Visiting `/` without a stored preference shows Chinese UI.
- The header exposes a visible `中文 / EN` control.
- Selecting `EN` changes platform UI labels to English without changing the URL.
- Reloading after selecting `EN` keeps English UI.
- Selecting `中文` restores Chinese UI and persists it.
- Catalog data is displayed exactly as stored in Convex.
- No Convex schema migration is introduced for this feature.
- Existing unit, Convex contract, build, and Playwright smoke tests pass after updates.
